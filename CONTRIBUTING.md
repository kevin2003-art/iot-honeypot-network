# Contributing Guide

## Branch Strategy
- Never commit directly to main
- All work goes on feature branches: feat/your-feature-name
- Bug fixes go on fix branches: fix/your-fix-name

## Commit Messages
Use semantic prefixes:
- feat: new feature
- fix: bug fix
- docs: documentation only
- refactor: code change with no new feature

## Pull Requests
- All branches merged via squash and merge
- PR title must match the commit message format
- Delete branch after merging
